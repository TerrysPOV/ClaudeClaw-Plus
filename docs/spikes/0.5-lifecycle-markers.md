# Spike 0.5 — Lifecycle markers in JSONL

Status: complete
Date: 2026-05-17
claude version: 2.1.143
Probes: `docs/spikes/probes/lifecycle-markers-probe.ts`, `clear-with-history-probe.ts`
Fixtures: `docs/spikes/fixtures/lifecycle/{compact,clear,quit}/`

Method: spawned claude via `bun-pty` (Spike 0.4 ruled plain stdin out), typed each slash into the REPL, diffed JSONL before/after. For `/clear` ran a second probe with a real first turn to disambiguate truncate vs rotate. Cross-referenced Spike 0.2 fixture 03 for real auto-compaction.

## /compact

- Marker line in JSONL? **Yes — two:** (1) `{type:"user", message.content:"<command-name>/compact</command-name>..."}` and (2) `{type:"system", subtype:"local_command", content:"<local-command-stderr>Error: Not enough messages to compact.</local-command-stderr>"}` (when compact can't run). When it succeeds (real history; Spike 0.2 fixture 03), claude additionally emits `{type:"system", subtype:"compact_boundary", content:"Conversation compacted", compactMetadata:{trigger, preTokens, postTokens, durationMs}}` — the authoritative compaction marker.
- File rewrite? **No.** JSONL appended, not truncated or rotated.
- stdout/stderr signal? TUI repaints inline; PTY stdout shows only Ink escapes.

## /clear

- Marker line in JSONL? **Yes, in a new file.** `<command-name>/clear</command-name>` envelope + empty `<local-command-stdout></local-command-stdout>` are written, but to a **new** JSONL — not the active one.
- File rewrite? **No — rotation.** With a real first turn (128 KB JSONL): pre-/clear `5af66010-...jsonl` 128 759 B; post-/clear same file 128 759 B (untouched); post-/exit old file unchanged **plus** new `deb2de28-...jsonl` (9 KB) holding the `/clear` + `/exit` envelopes. Old transcript is frozen; new session UUID allocated; new JSONL started. **No marker is written to the old file announcing rotation.**
- stdout/stderr signal? TUI repaints welcome banner. New session UUID shown in footer.

## /quit (alias /exit)

- Marker line in JSONL? **Yes.** `/quit` translates to `/exit`. Records `<command-name>/exit</command-name>` user envelope + `{type:"system", subtype:"local_command", content:"<local-command-stdout>Goodbye!</local-command-stdout>"}` (farewell string randomised — observed `Goodbye!` and `See ya!`).
- File rewrite? No. Final append, then file closed.
- stdout/stderr signal? PTY `onExit` fires with code 0. Stdout prints `Resume this session with: claude --resume <uuid>` immediately before exit.

## Markers absent — fallback signals available

Spec §5.2's `session.init` / `session.end` / `session.compact` **do not exist as top-level types.** Spike 0.2 confirmed across all `~/.claude/projects/*` <7d; this probe re-confirms.

| Lifecycle | Authoritative signal | Fallback |
|---|---|---|
| Init | First line in new `<sessionId>.jsonl` (no dedicated marker) | `fs.watch` `add` event on project dir |
| Compact (ok) | `{type:"system", subtype:"compact_boundary", compactMetadata:{...}}` | — |
| Compact (no-op) | `<command-name>/compact</command-name>` + `local-command-stderr` | — |
| Clear | New `<uuid>.jsonl` materialises in project dir | `fs.watch` `add` event |
| Quit / Exit | `<command-name>/exit</command-name>` envelope + PTY exit 0 | Process exit |

## Spec implications

- **§5.2: confirmed adjust.** Drop `session.init`/`session.end`/`session.compact` from the tailer schema. Use `compact_boundary` for successful compaction; `fs.watch` `add` on `~/.claude/projects/<cwd>/*.jsonl` for both init and `/clear` (both materialise as new files); PTY `onExit` for end. Attribute end-cause via the last `<command-name>/exit</command-name>` envelope (absent → crash).
- **`/clear` is rotation, not end.** Bus tailer treats it as `session.end(oldId) → session.init(newId)` and re-binds subscribers. Project dir is cwd-keyed and stable; one dir watcher catches it.
- **Failed slash commands are recorded** — useful for Policy Engine to detect `Error: Not enough messages to compact.` and suppress retry storms when daemon-triggered `/compact` hits a short session.
