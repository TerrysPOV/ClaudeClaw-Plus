# Spike 0.4 — Slash commands via stdin

Status: complete

## Plain stdin slash relay

- Accepted? **No.**
- Evidence: `Bun.spawn(["claude"], { stdin: 'pipe' })` never enters the interactive REPL. claude detects `process.stdin.isTTY === false`, downshifts to non-interactive `--print` mode, and demands a prompt:
  ```
  Warning: no stdin data received in 3s, proceeding without it. ...
  Error: Input must be provided either through stdin or as a prompt argument when using --print
  ```
  Exit code 1 in 12 s, 0 bytes on stdout. The `/compact\n` we wrote was discarded — never reached the slash parser.

## TTY slash relay (via bun-pty)

- Accepted? **Yes.**
- Evidence: same invocation through `bun-pty` paints the full TUI banner. `/compact\r` was processed (Ink re-render) and `/quit\r` exited cleanly (code 0, "Resume this session with: claude --resume ..."). The JSONL at `~/.claude/projects/.../<uuid>.jsonl` records `{type:"user", message.content:"<command-name>/compact</command-name>..."}` followed by `{type:"system", subtype:"local_command", content:"<local-command-stderr>Error: Not enough messages to compact.</local-command-stderr>"}`.

## isatty(0) check confirmed?

**Yes — binary-confirmed.** `strings /Users/terrenceyodaiken/.local/share/claude/versions/2.1.143` shows:
```
if (isFirst && process.stdin.isTTY)
if (printInitialMessage(!0), process.stdin.isTTY)
```
The `--help` text documents the gate: *"The workspace trust dialog is skipped when Claude is run in non-interactive mode (via -p, or when stdout is not a TTY)."* REPL is gated on both `stdin.isTTY` AND `stdout.isTTY`; a non-TTY on either side forces `--print`.

The spec §5.3 "write `/compact\n` to the supervised stdin" path is therefore fundamentally incompatible with claude. No no-op input loop can revive it — the REPL never starts.

## Recommendation

- **§5.3 default supervision mode: change `process` → `pty`.** Supervised claude needs a PTY on fd 0/1 to start the REPL. `bun-pty` is already a dependency (`src/runner/pty-process.ts`); migration cost is zero.
- **Drop the `process` mode** or rename it `headless` and restrict it to `claude -p` one-shots that never see slash commands. Anything receiving `/compact`, `/clear`, `/quit`, or a user slash command must be on a PTY.
- **tmux is NOT required on Unix.** `bun-pty` (Rust `portable_pty`) provides a real PTY without a multiplexer. tmux stays optional for users wanting detached terminals. Linux + macOS + Windows all work.

### Fallback chosen

Option (a) — PTY everywhere for supervised claude. Already proven in production via `src/runner/pty-process.ts`. The spec just needs the default flipped; no new fallback code.
