## Lifecycle markers — empirical finding

Tried: searched all JSONL under `~/.claude/projects` (mtime <7d) for types matching `session\.(init|end|compact)`.

Observed: **No such top-level types exist.** The only compact signal is `{type:"system", subtype:"compact_boundary", compactMetadata:{trigger,preTokens,postTokens,durationMs}}` — see fixture 03.

No `result` type in JSONL either — `result` is a stdout event from `claude -p --output-format=json`. `src/runner.ts:936-940` already treats it as stdout-only.

## Spec implications (§5.2)

1. **Drop `session.init`/`session.end`/`session.compact` from the tailer type table.** Detect compact via `{type:"system", subtype:"compact_boundary"}`. Session lifecycle is inferred (process exit / first non-empty line / inactivity).
2. **`tool_result` belongs under `user.message.content[]`**, not top-level. `src/runner.ts:923-934` already does this correctly — port verbatim into the Bus tailer.
3. Add `pr-link`, `agent-name`, `custom-title` to §5.2's type list (Nibbler missed these).
4. Treat `system` as a *container*, not a leaf. Wire subtypes: `compact_boundary` (mandatory), `turn_duration` (metrics), `stop_hook_summary`, `away_summary`, `scheduled_task_fire`.
5. **Fallback inference confirmed**: start = first non-empty line; end = file quiescent after process exit; compact = `compact_boundary`. No JSONL signal for "session.end" — process-exit watch is correct.
6. `tool_result.content` is **string OR array** (array when tool returned images). Tailer must `typeof === "string"` check before string ops.
7. Assistant messages may carry `error` / `isApiErrorMessage` / `apiErrorStatus` — surface as error events distinct from successful turns.
