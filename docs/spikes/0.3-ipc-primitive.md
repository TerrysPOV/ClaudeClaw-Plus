# Spike 0.3 — IPC primitive choice

Status: partial — Windows named pipe untested (no Windows host)
Date: 2026-05-17
Platform tested: darwin/arm64. Bun 1.3.4 (Node-compat 24.3.0).

Probes: `docs/spikes/probes/uds-probe.ts`, `docs/spikes/probes/tcp-probe.ts`. 100 samples/metric, p50/p99, 10-iter warmup.

## UDS results

**Path budget on macOS** (104B `sun_path` cap, NUL-inclusive). `$HOME=/Users/terrenceyodaiken` (23B), `XDG_RUNTIME_DIR` unset (macOS default). Template: `$HOME/.claudeclaw/run/bus-<8>-<agent>.sock`.

| agent slug | final bytes | bind |
|---|---|---|
| `triage-agent` | 70 | OK |
| `research-agent` | 72 | OK |
| `…-agent-instance` (42c) | 100 | **FAILS** at atomic-create |
| `…-agent-insta` (38c) | 97 | OK |

**Spec correction.** §5.4 says "fails fast if > 100B (4B margin)". Wrong: atomic-create binds `<path>.tmp` first, so `final + 4 + NUL ≤ 104`. Safe budget is **final ≤ 96B**, agent-slug max ~36 chars on typical macOS `$HOME`.

**Connect:** p50 = 0.039 ms · p99 = 0.190 ms. **1KB RTT (persistent):** p50 = 0.007 ms · p99 = 0.025 ms.

**Atomic create:** works. `.tmp` bind → `chmod 0600` → `rename`. Mode `0600`, no `.tmp` leftover.

**Crash recovery:** works. Stale-socket-after-crash → client `connect()` OS-rejected → probe-via-`connect()` cleanup unlinks + rebinds (rebind RTT = 0.053 ms).

## Localhost-TCP results

**Connect:** p50 = 0.078 ms · p99 = 1.470 ms. **1KB RTT (fresh-connect/request):** p50 = 0.051 ms · p99 = 0.180 ms.

**Token gating:** works. Bad token → framed `403`, server reject counter incremented. Token file `0600` at `~/.claudeclaw/agents/<id>/bus-token`. Bind enforced to `127.0.0.1`.

**Env-var visibility on macOS (verified empirically):** `ps eww -p <pid>` as the same user exposes the full environ including `CCAW_BUS_TOKEN`. Same threat model as Linux `/proc/<pid>/environ`, not weaker. The "macOS is stricter" assumption is wrong.

## Windows named pipe

**Known (Bun issue tracker, not tested):**
- Basic support landed via PR #13838 (Aug 2024). Bun 1.3.4 includes it.
- **No SDDL API exposed.** `node:net` does not surface libuv's named-pipe security-descriptor options — cannot restrict pipe to current user SID from Bun TS.
- **Open bug #30265:** busy-pipe `net.listen()` panics with internal assertion instead of `EADDRINUSE`. Blocks UDS-style collision handling.

**Needs Windows-host validation:** (1) bind succeeds for `\\\\.\\pipe\\claudeclaw-bus-<id>-<agent>`; (2) DACL behaviour from a second user; (3) #30265 crash blast radius; (4) RTT parity with UDS.

## Recommendation per platform

- **macOS:** UDS, corrected budget (final ≤ 96B).
- **Linux:** UDS. `XDG_RUNTIME_DIR` usually set (`/run/user/<uid>`, ~14B) → roomier budget.
- **Windows:** TCP+token as default. Named pipe is aspirational pending SDDL + #30265 validation.

## Spec implications

1. **§5.4 path budget:** "> 100B" → "`final + 4 (".tmp") > 100B`, i.e. final > 96B". Verified: 100B final fails because `.tmp` overflows the 104B cap.
2. **§5.4 Windows default:** demote named pipe from default to "preferred when validated". TCP+token is the safe default until SDDL + #30265 confirmed.
3. **§5.4 macOS env caveat:** add explicit note — `ps eww -p <pid>` exposes env to same-UID processes; macOS is not stricter than Linux.
4. **§5.4 Windows same-UID:** the "same user SID" analog cannot be enforced today via Bun — reinforces the TCP+token Windows default.
